# User Journey หลัก

ผู้ใช้เป้าหมาย: เจ้าของบ้านที่มีบิลค่าไฟ แต่ยังไม่มีความรู้ด้าน Load Profile หรืออัตรา TOU

1. เข้า `/analysis/new` แล้วเลือกเป้าหมาย: ลดค่าไฟ, ดูความเหมาะสม TOU, ดูความคุ้ม Solar หรือเข้าใจพฤติกรรมการใช้ไฟ
2. เลือกประเภทอาคาร/ผู้ใช้ และเลือกข้อมูลที่มี ระบบส่ง context นี้ไปยังหน้ากรอกข้อมูล
3. เพิ่มบิลค่าไฟย้อนหลัง หรือสร้าง/นำเข้ารูปแบบการใช้ไฟ ข้อมูลเดิมบันทึกด้วย storage helpers เดิม
4. ใช้ checklist ส่วนกลางเพื่อตรวจข้อมูลที่ขาดและย้อนกลับไปแก้ได้
5. เปรียบเทียบค่าไฟและ TOU หรือประเมิน Solar ตามข้อมูลที่พร้อม โดยไม่บังคับ Battery/EV
6. เปิดรายงานเมื่อผลและ data readiness พร้อม

## กรณีไม่มี Load Profile

ระบบเสนอ “สร้างจากเครื่องใช้ไฟฟ้า” เป็นทางเลือกหลัก และ “นำเข้าไฟล์” เป็นทางเลือกสำหรับผู้มี Smart Meter ผู้ใช้ยังเพิ่มบิลก่อนเพื่อเห็นภาพค่าใช้จ่ายได้ จึงไม่มี dead end ที่บังคับให้มีไฟล์มิเตอร์

## หลักการตัดสินใจ

- ค่าเริ่มต้นของเส้นทางคือข้อมูลจริงจากบิลหรือรายการเครื่องใช้ ไม่สร้างผลจริงจากตัวอย่าง
- ปุ่ม “ดูตัวอย่างผลลัพธ์” สร้าง workspace โหมด sample อย่างชัดเจน และผลที่ตามมายังคงแยก sample/user ตาม logic เดิม
- Checklist แสดงเฉพาะสถานะจากข้อมูลที่บันทึกจริง ไม่ตั้งว่าขั้นตอนเสร็จหากไม่มีข้อมูล
